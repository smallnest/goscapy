Package Structure

goscapy is organized into focused packages:

PackageImport PathDescription
goscapypkg/goscapyBuilder API and shortcut functions — the main entry point
packetpkg/packetCore Packet/Layer types, Build, Dissect, field binding, protocol registration
layerspkg/layersProtocol layer definitions (Ethernet, ARP, IP, TCP, UDP, ICMP, IPv6, Dot1Q, VXLAN, GRE, DNS, DHCP, Raw)
fieldspkg/fieldsField type system — serialization, deserialization, TLV options
sendrecvpkg/sendrecvRaw socket send/receive (Send, Sendp, Recv, SendRecv)
sniffpkg/sniffPacket sniffing with BPF filter support
💡 Tip
Most users only need goscapy (builders + shortcuts) and packet (dissect + send). Import the others as needed.

Builder API Reference

Every protocol has a Builder with fluent setter methods. Chain layers with .Over(), finalize with .Build().

go
raw, err := goscapy.NewEthernet().
    SrcMAC("...").DstMAC("...").
    Over(goscapy.NewIP().SrcIP("...").DstIP("...")).
    Over(goscapy.NewTCP().SrcPort(80).DstPort(8080)).
    Build()

Layer Binding

When you stack layers with .Over(), goscapy automatically infers fields between adjacent layers. For example, placing IP over Ethernet automatically sets EtherType = 0x0800.

go
// EtherType is auto-set to 0x0800 (IPv4)
// IP.Proto is auto-set to 1 (ICMP)
raw, _ := goscapy.NewEthernet().
    SrcMAC("aa:bb:cc:dd:ee:ff").
    DstMAC("ff:ff:ff:ff:ff:ff").
    Over(goscapy.NewIP().SrcIP("192.168.1.1").DstIP("8.8.8.8")).
    Over(goscapy.NewICMP().Type(8).Code(0)).
    Build()
📝 Note
Bindings are registered during package initialization. You can also register custom bindings with packet.RegisterBinding(upper, lower, field, value).

Protocol Registration

goscapy uses a registration system for extensible protocol support. All built-in protocols register themselves during init().

Key Field

A key field identifies which upper-layer protocol follows. For IP, the key field is proto.

go
packet.RegisterKeyField("IP", "proto")

Next-Layer Mapping

go
packet.RegisterNextLayer("IP", 6, "TCP")   // proto=6 → TCP
packet.RegisterNextLayer("IP", 17, "UDP")  // proto=17 → UDP

Heuristic Dissectors

go
// DHCP detected by UDP ports 67/68
packet.RegisterHeuristic("UDP", "dport", 67, "DHCP")
packet.RegisterHeuristic("UDP", "dport", 68, "DHCP")

Tunnel Payload

go
// VXLAN carries inner Ethernet frames
packet.RegisterTunnelPayload("VXLAN", "Ethernet")
💡 Tip
The dissection pipeline supports up to 8 levels of nested tunnel encapsulation.

Build Hooks

Build hooks run custom logic during serialization. All built-in checksums use build hooks internally.

go
// Register a custom build hook
packet.RegisterBuildHook("IP", func(pkt *packet.Packet, idx int, upper []byte) ([]byte, error) {
    layer := pkt.Layers()[idx]
    layer.Set("len", uint16(len(upper)+20))
    return layer.BuildBytes()
})

Dissector Pipeline

The Dissect function parses raw bytes into a layered Packet:

  1. Calling the start function to identify the first protocol
  2. Running the protocol's dissector to extract fields and determine header size
  3. Looking up the key field to find the next protocol
  4. Falling back to heuristic rules if no key-field match
  5. Recursing into tunnel payloads (e.g. VXLAN → inner Ethernet)
  6. Stopping when no more layers can be identified
go
// Dissect starting from Ethernet
pkt, err := packet.Dissect(raw, packet.DissectEthernet)

// Or from IP directly
pkt, err := packet.DissectByProto(raw, "IP")

// Access layers
pkt.String()              // "Ethernet / IP / TCP"
pkt.GetLayer("IP")        // get IP layer
pkt.HasLayer("TCP")       // true

Send & Receive

FunctionLevelDescription
Send(pkt, iface)L3Send at IP level; OS adds Ethernet header
Sendp(pkt, iface)L2Send full Ethernet frame; you control everything
Recv(iface, timeout)L2Receive one packet and close
SendRecv(pkt, iface, timeout)L3+L2Send then collect all responses
SendRecv1(pkt, iface, timeout)L3+L2Send then return first response
📝 Note
SendRecv opens the receiver before sending to avoid missing fast responses on loopback.

Raw Socket Wrapper (RawConn)

For scenarios where you need direct access to a raw socket without manually handling file descriptors, address translation, and timeout logic, goscapy provides the RawConn object.

go
// Create a raw connection (1=ICMP, 6=TCP, 17=UDP)
conn, err := sendrecv.DialRaw(1)
if err != nil {
    log.Fatal(err)
}
defer conn.Close()

// Send raw payload
err = conn.Send(data, "127.0.0.1")

// Receive payload and source IP with timeout
reply, srcIP, err := conn.Recv(3 * time.Second)

For quick, one-off raw socket transactions, you can use the package-level convenience functions:

go
// Send payload directly without maintaining connection state
err := sendrecv.SendRaw(1, data, "127.0.0.1")

// Receive payload directly with timeout
reply, srcIP, err := sendrecv.RecvRaw(1, 3 * time.Second)

For high-performance scenarios (such as scanning or bulk pinging), you can use the BatchConn API to send/receive multiple packets in a single system call (utilizing sendmmsg and recvmmsg on Linux, falling back to a sequential loop on macOS):

go
// Get a BatchConn from an existing RawConn
batch := conn.Batch()

// Batch send multiple messages
msgs := []sendrecv.BatchMsg{
    {Data: payload1, Dst: "127.0.0.1"},
    {Data: payload2, Dst: "127.0.0.1"},
}
nSent, err := batch.SendBatch(msgs)

// Batch receive multiple messages
results, err := batch.RecvBatch(2, 3 * time.Second)

For sending large payloads without kernel buffer copying overhead, you can enable ZeroCopy. This sets the SO_ZEROCOPY socket option and makes Send use the MSG_ZEROCOPY flag on Linux (directly returning ErrNotSupported on macOS):

go
// Enable ZeroCopy mode
err := conn.SetZeroCopy(true)

// Send raw payloads
err = conn.Send(largePayload, "127.0.0.1")

// Wait for all zero-copy transmissions to be completed by the kernel
ctx, cancel := context.WithTimeout(context.Background(), 5 * time.Second)
defer cancel()
err = conn.WaitZeroCopyCompletion(ctx)

For extreme performance and zero kernel context switching overhead, you can use the Linux io_uring-backed UringConn API. This provides asynchronous packet transmission and batch operations using kernel SQ/CQ queues (returns ErrNotSupported on macOS):

go
// Dial a raw socket connection backed by io_uring
conn, err := sendrecv.DialUringRaw(1) // 1 = IPPROTO_ICMP

// Submit an asynchronous send and obtain its operation ID
opID, err := conn.Send(payload, "127.0.0.1")

// Wait for completion queue event (CQE) with timeout
reply, src, err := conn.Recv(3 * time.Second)

// Perform batch send and receive operations in a single system call
results, err := conn.SendRecvBatch(msgs)

Sniffing

Callback API

go
err := sniff.Sniff(sniff.SniffConfig{
    Iface:   "eth0",
    Filter:  "icmp",
    Timeout: 10 * time.Second,
}, func(pkt *packet.Packet) bool {
    fmt.Println(pkt.String())
    return true // true = continue
})

Channel API

go
ch, stop := sniff.SniffChan(sniff.SniffConfig{
    Iface: "eth0",
    Count: 10,
})
defer stop()
for pkt := range ch {
    fmt.Println(pkt)
}

PacketMMAP (TPACKET_V3) API

For high-performance packet capture scenarios on Linux, you can utilize the memory-mapped ring buffer interface (TPACKET_V3). This bypasses BPF overhead and achieves much higher throughput (returns ErrNotSupported on macOS):

go
// Initialize PacketMMAP on a given interface
m, err := sendrecv.NewPacketMMAP("eth0")
defer m.Close()

// Read next packet (blocking with timeout)
pkt, err := m.Recv(1 * time.Second)

// Batch read packets
pkts, err := m.RecvBatch(10, 2 * time.Second)

// Retrieve capture statistics
stats := m.Stats()
fmt.Printf("Received: %d | Dropped: %d", stats.Received, stats.Dropped)

BPF Filters

BPF expressions are compiled to kernel-level filters for maximum performance. Requires tcpdump on PATH for compilation.

go
// Compile a filter string to BPF instructions
instructions, err := sniff.CompileFilter("tcp port 80 and host 192.168.1.1")

// Use pre-compiled instructions (bypasses tcpdump dependency)
ch, stop := sniff.SniffChan(sniff.SniffConfig{
    Iface:        "eth0",
    Instructions: instructions,
    Count:        100,
})
💡 Tip
Pre-compile filters with CompileFilter at startup to avoid runtime dependency on tcpdump.
