Send — 发送 ICMP 回显请求（L3）

构建 IP+ICMP 数据包并通过 L3 发送。操作系统会自动添加以太网头。Send() 是即发即忘模式 —— 不会等待响应。

🔨 Builder API

go
import (
    "fmt"
    "github.com/smallnest/goscapy/pkg/goscapy"
    "github.com/smallnest/goscapy/pkg/packet"
    "github.com/smallnest/goscapy/pkg/sendrecv"
)

// 构建 IP + ICMP 回显请求包（L3 — 无以太网层）
pkt, err := goscapy.NewIP().
    SrcIP("127.0.0.1").
    DstIP("127.0.0.1").
    Over(goscapy.NewICMP().
        Type(8).
        Code(0).
        ID(0x0001).
        Seq(1)).
    Packet()

if err != nil {
    fmt.Fatal(err)
}

// 在回环接口上通过 L3 发送
iface := sendrecv.LoopbackName() // macOS 上为 "lo0"，Linux 上为 "lo"
err = sendrecv.Send(pkt, iface)
if err != nil {
    fmt.Fatal(err)
}
fmt.Println("数据包已发送！")

⚡ 快捷函数

go
// 使用 Builder .Packet() 方法快速构建并获取 Packet
pkt, _ := goscapy.NewIP().
    SrcIP("127.0.0.1").
    DstIP("127.0.0.1").
    Over(goscapy.NewICMP().Type(8).Code(0)).
    Packet()
sendrecv.Send(pkt, sendrecv.LoopbackName())
📝 注意
Send() 工作在 L3（IP 层）—— 操作系统负责 L2 组帧（以太网头）。 如果数据包包含以太网层，构建时会自动跳过。 需要 root 权限（sudo）或 Linux 上的 CAP_NET_RAW 能力。

Recv — 接收一个数据包

在指定接口上打开接收器，等待一个数据包到达后关闭。Recv() 会阻塞直到数据包到达或超时。使用 errors.Is(err, sendrecv.ErrTimeout) 检测超时。

🔨 示例

go
import (
    "errors"
    "fmt"
    "time"

    "github.com/smallnest/goscapy/pkg/packet"
    "github.com/smallnest/goscapy/pkg/sendrecv"
)

iface := sendrecv.LoopbackName()

// 接收一个数据包，超时 5 秒
pkt, err := sendrecv.Recv(iface, 5*time.Second)
if err != nil {
    if errors.Is(err, sendrecv.ErrTimeout) {
        fmt.Println("超时未收到数据包")
    } else {
        fmt.Fatal(err)
    }
    return
}

// 打印接收到的数据包层信息
fmt.Println(pkt.String()) // 例如 "Ethernet / IP / ICMP"

// 访问 IP 层字段
ip := packet.IP(pkt)
if ip != nil {
    fmt.Printf("源: %s → 目标: %s\n", ip.SrcIP(), ip.DstIP())
}
💡 提示
如需接收多个数据包，请直接使用 OpenReceiver() 并在循环中调用 Recv()。 这样可以避免每个数据包都打开和关闭原始套接字。

🔧 持续接收循环

go
// 打开持久接收器以捕获多个数据包
rx, err := sendrecv.OpenReceiver(sendrecv.LoopbackName())
if err != nil {
    fmt.Fatal(err)
}
defer rx.Close()

// 最多读取 10 个数据包
for i := 0; i < 10; i++ {
    pkt, err := rx.Recv(3 * time.Second)
    if err != nil {
        if errors.Is(err, sendrecv.ErrTimeout) {
            fmt.Println("超时，停止接收。")
            break
        }
        fmt.Fatal(err)
    }
    fmt.Printf("数据包 %d: %s\n", i+1, pkt.String())
}

SendRecv1 — ICMP Ping

构建 ICMP 回显请求，通过 L3 发送并接收第一个响应。SendRecv1() 非常适合请求/响应协议（如 ICMP ping）—— 它只返回第一个匹配的回复。

🔨 Builder API

go
import (
    "fmt"
    "time"

    "github.com/smallnest/goscapy/pkg/goscapy"
    "github.com/smallnest/goscapy/pkg/layers"
    "github.com/smallnest/goscapy/pkg/packet"
    "github.com/smallnest/goscapy/pkg/sendrecv"
)

// 构建 IP + ICMP 回显请求
pkt, err := goscapy.NewIP().
    SrcIP("127.0.0.1").
    DstIP("127.0.0.1").
    Over(goscapy.NewICMP().
        Type(layers.ICMPEchoRequest).
        Code(0).
        ID(0x0001).
        Seq(1)).
    Packet()
if err != nil {
    fmt.Fatal(err)
}

// 发送并等待第一个响应（3 秒超时）
sent, resp, err := sendrecv.SendRecv1(
    pkt,
    sendrecv.LoopbackName(),
    3*time.Second,
)
if err != nil {
    fmt.Fatal(err)
}

fmt.Printf("已发送:     %s\n", sent.String())
if resp != nil {
    fmt.Printf("响应: %s\n", resp.String())

    // 检查响应是否为 ICMP 回显回复
    icmp := packet.ICMP(resp)
    if icmp != nil && icmp.Type() == layers.ICMPEchoReply {
        fmt.Println("✓ 收到 ICMP 回显回复！")
    }
}

⚡ 快捷函数

go
// 使用 Builder .Packet() 方法快速 ping
pkt, _ := goscapy.NewIP().
    SrcIP("127.0.0.1").
    DstIP("127.0.0.1").
    Over(goscapy.NewICMP().Type(8).Code(0)).
    Packet()

_, resp, _ := sendrecv.SendRecv1(pkt, sendrecv.LoopbackName(), 3*time.Second)
if resp != nil {
    fmt.Println("Ping 回复:", resp.String())
}
📝 注意
SendRecv1() 在发送之前就打开接收器，以避免错过快速响应 （例如在回环接口上，响应可能在顺序打开的接收器就绪之前就已到达）。

SendRecv — 收集所有响应

SendRecv() 通过 L3 发送数据包，并在超时窗口内收集所有响应数据包。当你预期多个响应（如广播查询）或想观察数据包触发的所有流量时非常有用。

🔨 示例

go
import (
    "fmt"
    "time"

    "github.com/smallnest/goscapy/pkg/goscapy"
    "github.com/smallnest/goscapy/pkg/layers"
    "github.com/smallnest/goscapy/pkg/sendrecv"
)

// 构建 ICMP 回显请求
pkt, _ := goscapy.NewIP().
    SrcIP("127.0.0.1").
    DstIP("127.0.0.1").
    Over(goscapy.NewICMP().
        Type(layers.ICMPEchoRequest).
        Code(0).
        ID(0x0001).
        Seq(1)).
    Packet()

// 发送并在 2 秒内收集所有响应
sent, responses, err := sendrecv.SendRecv(
    pkt,
    sendrecv.LoopbackName(),
    2*time.Second,
)
if err != nil {
    fmt.Fatal(err)
}

fmt.Printf("已发送: %s\n", sent.String())
fmt.Printf("收到 %d 个响应:\n", len(responses))
for i, resp := range responses {
    fmt.Printf("  [%d] %s\n", i+1, resp.String())
}

TCP SYN — 发送与接收

构造 TCP SYN 数据包，用 Send() 发送，然后用 Recv() 等待 SYN-ACK 响应。此示例展示了 TCP 端口扫描的两步发送-接收模式。

🔨 Builder API

go
import (
    "fmt"
    "time"

    "github.com/smallnest/goscapy/pkg/goscapy"
    "github.com/smallnest/goscapy/pkg/layers"
    "github.com/smallnest/goscapy/pkg/packet"
    "github.com/smallnest/goscapy/pkg/sendrecv"
)

// 构建 IP + TCP SYN 数据包
pkt, err := goscapy.NewIP().
    SrcIP("127.0.0.1").
    DstIP("127.0.0.1").
    Over(goscapy.NewTCP().
        SrcPort(12345).
        DstPort(80).
        Flags(layers.TCPSyn).
        Seq(1000)).
    Packet()
if err != nil {
    fmt.Fatal(err)
}

iface := sendrecv.LoopbackName()

// 第 1 步：发送 SYN 数据包
err = sendrecv.Send(pkt, iface)
if err != nil {
    fmt.Fatal(err)
}
fmt.Println("SYN 已发送到 127.0.0.1:80")

// 第 2 步：等待响应
resp, err := sendrecv.Recv(iface, 5*time.Second)
if err != nil {
    fmt.Fatal(err)
}

// 检查响应是否为 TCP
tcp := packet.TCP(resp)
if tcp != nil {
    flags := tcp.Flags()
    if flags&layers.TCPSyn != 0 && flags&layers.TCPAck != 0 {
        fmt.Printf("端口 %d: 开放（收到 SYN-ACK）\n", tcp.SrcPort())
    } else if flags&layers.TCPRst != 0 {
        fmt.Printf("端口 %d: 关闭（收到 RST）\n", tcp.SrcPort())
    }
}

⚡ 快捷函数

go
// 使用 Builder 快速构建 TCP SYN 包
pkt, _ := goscapy.NewIP().
    SrcIP("127.0.0.1").
    DstIP("127.0.0.1").
    Over(goscapy.NewTCP().SrcPort(12345).DstPort(80).Flags(layers.TCPSyn)).
    Packet()
sendrecv.Send(pkt, sendrecv.LoopbackName())
📝 注意
对于 TCP SYN 扫描，SendRecv1() 通常比单独的 Send() + Recv() 更方便，因为它在发送之前就打开接收器， 确保不会错过快速响应。

错误处理

生产环境代码应处理常见错误：超时、接口不存在、权限不足。以下是一个适用于所有发送/接收操作的健壮错误处理模式。

🔧 超时与错误检测

go
import (
    "errors"
    "fmt"
    "time"

    "github.com/smallnest/goscapy/pkg/goscapy"
    "github.com/smallnest/goscapy/pkg/sendrecv"
)

// 构建数据包
pkt, err := goscapy.NewIP().
    SrcIP("127.0.0.1").
    DstIP("127.0.0.1").
    Over(goscapy.NewICMP().
        Type(8).
        Code(0)).
    Packet()
if err != nil {
    fmt.Fatal("构建错误:", err)
}

// SendRecv1 完整错误处理
_, resp, err := sendrecv.SendRecv1(pkt, sendrecv.LoopbackName(), 3*time.Second)
switch {
case err == nil && resp != nil:
    fmt.Println("响应:", resp.String())
case err == nil && resp == nil:
    fmt.Println("已发送但超时内未收到响应")
case errors.Is(err, sendrecv.ErrTimeout):
    fmt.Println("操作超时")
default:
    fmt.Println("错误:", err)
    // 常见错误:
    //   - "sendrecv: open receiver: ..." → 接口不存在
    //   - "operation not permitted"     → 需要 root/sudo
    //   - "no such device"             → 错误的接口名
}
💡 提示
  • 始终使用 errors.Is(err, sendrecv.ErrTimeout) —— 不要用 == 比较。
  • 使用 sendrecv.LoopbackName() 编写跨平台代码，macOS 上为 lo0，Linux 上为 lo。
  • 原始套接字操作需要 root 权限。使用 sudo 运行或在 Linux 上设置 CAP_NET_RAW。
