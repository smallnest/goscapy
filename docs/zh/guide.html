包结构总览

goscapy 按功能划分为专注的包：

包导入路径说明
goscapypkg/goscapyBuilder API 和快捷函数 — 主入口
packetpkg/packet核心 Packet/Layer 类型、Build、Dissect、字段绑定、协议注册
layerspkg/layers协议层定义（Ethernet、ARP、IP、TCP、UDP、ICMP、IPv6、Dot1Q、VXLAN、GRE、DNS、DHCP、Raw）
fieldspkg/fields字段类型系统 — 序列化、反序列化、TLV 选项
sendrecvpkg/sendrecv原始套接字发送/接收（Send、Sendp、Recv、SendRecv）
sniffpkg/sniff数据包嗅探，支持 BPF 过滤器
💡 提示
大多数用户只需要 goscapy（构建器 + 快捷函数）和 packet（解析 + 发送）。其他包按需导入。

Builder API 参考

每个协议都有一个 Builder，提供流畅的 setter 方法。使用 .Over() 串联各层，最后调用 .Build() 完成构建。

go
raw, err := goscapy.NewEthernet().
    SrcMAC("...").DstMAC("...").
    Over(goscapy.NewIP().SrcIP("...").DstIP("...")).
    Over(goscapy.NewTCP().SrcPort(80).DstPort(8080)).
    Build()

层间绑定

使用 .Over() 串联各层时，goscapy 会自动推断相邻层之间的字段。例如，将 IP 叠加到以太网上会自动设置 EtherType = 0x0800。

go
// EtherType 自动设置为 0x0800（IPv4）
// IP.Proto 自动设置为 1（ICMP）
raw, _ := goscapy.NewEthernet().
    SrcMAC("aa:bb:cc:dd:ee:ff").
    DstMAC("ff:ff:ff:ff:ff:ff").
    Over(goscapy.NewIP().SrcIP("192.168.1.1").DstIP("8.8.8.8")).
    Over(goscapy.NewICMP().Type(8).Code(0)).
    Build()
📝 注意
绑定在包初始化期间注册。你也可以使用 packet.RegisterBinding(upper, lower, field, value) 注册自定义绑定。

协议注册系统

goscapy 使用注册系统实现可扩展的协议支持。所有内置协议在 init() 期间自动注册。

Key Field（关键字段）

Key field 用于标识上层协议。对于 IP，key field 是 proto。

go
packet.RegisterKeyField("IP", "proto")

Next-Layer Mapping（下一层映射）

go
packet.RegisterNextLayer("IP", 6, "TCP")   // proto=6 → TCP
packet.RegisterNextLayer("IP", 17, "UDP")  // proto=17 → UDP

Heuristic Dissectors（启发式解析器）

go
// DHCP 通过 UDP 端口 67/68 检测
packet.RegisterHeuristic("UDP", "dport", 67, "DHCP")
packet.RegisterHeuristic("UDP", "dport", 68, "DHCP")

Tunnel Payload（隧道载荷）

go
// VXLAN 携带内部以太网帧
packet.RegisterTunnelPayload("VXLAN", "Ethernet")
💡 提示
解析管线支持最多 8 层嵌套隧道封装。

自动校验和

IP、TCP、UDP、ICMP、ICMPv6 和 IPv6 的校验和在 .Build() 期间通过构建钩子自动计算。

协议自动计算的字段
IP总长度、头部校验和
TCP校验和（含伪头部）
UDP长度、校验和（含伪头部）
ICMP校验和
IPv6载荷长度
ICMPv6校验和（含 IPv6 伪头部）

构建钩子

构建钩子在序列化期间运行自定义逻辑。所有内置的校验和计算都通过构建钩子实现。

go
// 注册自定义构建钩子
packet.RegisterBuildHook("IP", func(pkt *packet.Packet, idx int, upper []byte) ([]byte, error) {
    layer := pkt.Layers()[idx]
    layer.Set("len", uint16(len(upper)+20))
    return layer.BuildBytes()
})

解析器管线

Dissect 函数通过以下步骤将原始字节解析为分层 Packet：

  1. 调用起始函数识别第一个协议
  2. 运行协议的解析器提取字段并确定头部大小
  3. 查找 key field 以确定下一个协议
  4. 如果没有 key-field 匹配，则回退到启发式规则
  5. 递归进入隧道载荷（如 VXLAN → 内部 Ethernet）
  6. 当无法识别更多层时停止
go
// 从 Ethernet 开始解析
pkt, err := packet.Dissect(raw, packet.DissectEthernet)

// 或直接从 IP 开始
pkt, err := packet.DissectByProto(raw, "IP")

// 访问各层
pkt.String()              // "Ethernet / IP / TCP"
pkt.GetLayer("IP")        // 获取 IP 层
pkt.HasLayer("TCP")       // true

发送与接收

函数层级说明
Send(pkt, iface)L3在 IP 层级发送；OS 添加以太网头
Sendp(pkt, iface)L2发送完整以太网帧；你控制一切
Recv(iface, timeout)L2接收一个数据包并关闭
SendRecv(pkt, iface, timeout)L3+L2发送后收集所有响应
SendRecv1(pkt, iface, timeout)L3+L2发送后返回第一个响应
📝 注意
SendRecv 在发送之前打开接收器，以避免在回环接口上错过快速响应。

原始套接字封装 (RawConn)

对于需要直接访问原始套接字，又不想手动处理文件描述符、地址转换以及 select/超时等底层细节的场景，goscapy 提供了 RawConn 对象。

go
// 创建原始套接字连接 (1 为 ICMP，6 为 TCP，17 为 UDP)
conn, err := sendrecv.DialRaw(1)
if err != nil {
    log.Fatal(err)
}
defer conn.Close()

// 发送原始载荷
err = conn.Send(data, "127.0.0.1")

// 接收载荷和来源 IP（支持超时）
reply, srcIP, err := conn.Recv(3 * time.Second)

对于快速的一次性套接字操作，你可以使用包级别的便捷函数：

go
// 直接发送载荷，无需维护连接状态
err := sendrecv.SendRaw(1, data, "127.0.0.1")

// 直接接收载荷（支持超时）
reply, srcIP, err := sendrecv.RecvRaw(1, 3 * time.Second)

对于高并发/高性能场景（如扫描或批量 ping 探测），你可以使用 BatchConn API 在单次系统调用中发送/接收多个报文（在 Linux 上利用 sendmmsg 和 recvmmsg，在 macOS 上自动降级为循环处理）：

go
// 从现有 RawConn 取得 BatchConn
batch := conn.Batch()

// 批量发送多个报文
msgs := []sendrecv.BatchMsg{
    {Data: payload1, Dst: "127.0.0.1"},
    {Data: payload2, Dst: "127.0.0.1"},
}
nSent, err := batch.SendBatch(msgs)

// 批量接收多个报文
results, err := batch.RecvBatch(2, 3 * time.Second)

对于发送大报文（如巨型帧或大文件传输）且希望避免内核拷贝开销的场景，你可以开启 ZeroCopy。该操作在 Linux 上会设置 SO_ZEROCOPY 套接字选项，并使 Send 时自动使用 MSG_ZEROCOPY 标志（在 macOS 上会直接返回 ErrNotSupported）：

go
// 启用零拷贝模式
err := conn.SetZeroCopy(true)

// 发送大原始载荷
err = conn.Send(largePayload, "127.0.0.1")

// 等待所有零拷贝报文被内核传输完毕并释放用户态内存
ctx, cancel := context.WithTimeout(context.Background(), 5 * time.Second)
defer cancel()
err = conn.WaitZeroCopyCompletion(ctx)

嗅探

回调 API

go
err := sniff.Sniff(sniff.SniffConfig{
    Iface:   "eth0",
    Filter:  "icmp",
    Timeout: 10 * time.Second,
}, func(pkt *packet.Packet) bool {
    fmt.Println(pkt.String())
    return true // true = 继续嗅探
})

通道 API

go
ch, stop := sniff.SniffChan(sniff.SniffConfig{
    Iface: "eth0",
    Count: 10,
})
defer stop()
for pkt := range ch {
    fmt.Println(pkt)
}

BPF 过滤器

BPF 表达式被编译为内核级过滤器，实现最高性能。编译需要系统 PATH 中有 tcpdump。

go
// 将过滤器字符串编译为 BPF 指令
instructions, err := sniff.CompileFilter("tcp port 80 and host 192.168.1.1")

// 使用预编译的指令（无需 tcpdump）
ch, stop := sniff.SniffChan(sniff.SniffConfig{
    Iface:        "eth0",
    Instructions: instructions,
    Count:        100,
})
💡 提示
在启动时使用 CompileFilter 预编译过滤器，避免运行时依赖 tcpdump。
