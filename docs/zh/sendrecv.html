平台兼容性矩阵

sendrecv 包在提供标准原始套接字的同时，还针对 Linux 系统进行了高度优化的深层次高性能网络收发特征封装。

功能 / API macOS (Darwin) Linux 权限要求与内核选项
RawConn (DialRaw) ✅ 支持 ✅ 支持 需要 Root 权限以开启原始套接字
SendRaw / RecvRaw ✅ 支持 ✅ 支持 包级别的无状态便捷调用接口；需要 Root 权限
BatchConn (SendBatch / RecvBatch) ⚠️ 顺序降级模拟 ✅ 支持 在 Linux 下利用 sendmmsg 与 recvmmsg 系统调用减少切换
ZeroCopy (SetZeroCopy / MSG_ZEROCOPY) ❌ 返回 ErrNotSupported ✅ 支持 要求 Linux 内核 4.14+；开启 SO_ZEROCOPY 选项
UringConn (io_uring 异步 IO) ❌ 返回 ErrNotSupported ✅ 支持 要求 Linux 内核 5.1+；自动映射 SQ/CQ 双环缓冲区
PacketMMAP (TPACKET_V3 捕获) ❌ 返回 ErrNotSupported ✅ 支持 需要 Linux 特有的 AF_PACKET 链路层捕获机制

原始套接字连接 (RawConn)

RawConn 负责管理长连接原始套接字。通过 DialRaw 接口进行创建。

DialRaw

go
func DialRaw(proto int) (*RawConn, error)

针对指定的 IP 协议号（如 1 表示 ICMP, 6 表示 TCP, 17 表示 UDP）建立长连接原始套接字。如果由于权限不足等原因导致套接字创建失败，则返回 error。

Send

go
func (c *RawConn) Send(payload []byte, dst string) error

向目标 IP 发送原始载荷数据（无需包含以太网帧头）。如果开启了零拷贝模式，该发送会异步标记对应载荷内存，直到内核完成发送。

Recv

go
func (c *RawConn) Recv(timeout time.Duration) ([]byte, string, error)

阻塞式接收报文，直至报文到达或超时。返回报文载荷字节流、源端 IP 地址和错误信息。超时将返回 sendrecv.ErrTimeout。

Batch

go
func (c *RawConn) Batch() *BatchConn

返回绑定于当前原始连接的 BatchConn 实例，用于批量发送与接收报文。

SetZeroCopy

go
func (c *RawConn) SetZeroCopy(enable bool) error

启用或禁用零拷贝发送模式。在 Linux 下会调用 SO_ZEROCOPY 套接字选项；在 macOS 下会返回 sendrecv.ErrNotSupported 错误。

WaitZeroCopyCompletion

go
func (c *RawConn) WaitZeroCopyCompletion(ctx context.Context) error

同步等待 Linux 内核完全释放所有未完成的零拷贝发送内存。如果上下文超时仍未完成则返回 error。

Close

go
func (c *RawConn) Close() error

关闭原始套接字文件描述符，释放系统资源。

无状态便捷接口

对于无需复用连接的简单单次调用，可以使用以下包级别全局函数：

SendRaw

go
func SendRaw(proto int, payload []byte, dst string) error

临时创建原始套接字，向指定目标地址发送数据并立即关闭套接字。

RecvRaw

go
func RecvRaw(proto int, timeout time.Duration) ([]byte, string, error)

临时创建原始套接字，阻塞式接收报文直至超时或报文到达，随后关闭套接字。

批量发送与接收 (BatchConn)

BatchConn 利用 Linux 的 sendmmsg 与 recvmmsg 系统调用进行批量读写，显著减少用户态与内核态之间的上下文切换次数。

SendBatch

go
func (b *BatchConn) SendBatch(msgs []BatchMsg) (int, error)

批量发送多个载荷包。在 macOS 下会自动降级为顺序发送循环。返回成功发送的报文数量。

go
type BatchMsg struct {
    Data []byte
    Dst  string
}

RecvBatch

go
func (b *BatchConn) RecvBatch(n int, timeout time.Duration) ([]BatchResult, error)

阻塞接收最多 n 个报文，直至超时。返回 BatchResult 数组。

go
type BatchResult struct {
    Data  []byte
    SrcIP string
    Err   error
}

io_uring 异步套接字 (UringConn)

UringConn 基于现代 Linux 的 io_uring 异步 I/O 架构。通过将读写请求推送到提交队列（SQ）并轮询完成队列（CQ）来取得极致吞吐量，规避系统调用中断消耗。

DialUringRaw

go
func DialUringRaw(proto int) (*UringConn, error)

在 Linux 上初始化绑定 `io_uring` 原始队列的套接字连接。在 macOS 或版本低于 5.1 的 Linux 内核上返回 sendrecv.ErrNotSupported 错误。

Send

go
func (u *UringConn) Send(payload []byte, dst string) (uint64, error)

向 SQ 提交一个非阻塞的原始报文写请求。返回一个唯一的任务操作 ID，供以后轮询 CQ 时识别。

Recv

go
func (u *UringConn) Recv(timeout time.Duration) ([]byte, string, error)

轮询 CQ 完成队列，等待并在超时范围内提取最近读到的数据。

SendRecvBatch

go
func (u *UringConn) SendRecvBatch(msgs []BatchMsg) ([]BatchResult, error)

单次系统调用向内核环提交一批并行的读与写任务。

Close

go
func (u *UringConn) Close() error

销毁 SQ/CQ 映射区域并安全地关闭对应的原始套接字句柄。

环形缓冲区报文捕获 (PacketMMAP)

PacketMMAP 在 Linux 上通过映射一个零拷贝环形缓冲区（TPACKET_V3）与 AF_PACKET 链路层套接字相连，实现高性能的报文侦听抓包。

NewPacketMMAP

go
func NewPacketMMAP(iface string) (*PacketMMAP, error)

在指定网卡接口（例如 eth0）上创建并映射 AF_PACKET 链路层捕获队列。在 macOS 上会返回 sendrecv.ErrNotSupported 错误。

Recv

go
func (m *PacketMMAP) Recv(timeout time.Duration) (*packet.Packet, error)

从共享环缓冲区内存中收取下一个数据帧，并自动从链路层协议开始利用 goscapy 解析器进行拆包处理。

RecvBatch

go
func (m *PacketMMAP) RecvBatch(n int, timeout time.Duration) ([]*packet.Packet, error)

一次性从缓存环中批量取出并解析最多 n 个数据包。

Stats

go
func (m *PacketMMAP) Stats() PacketMMAPStats

直接提取底层的实时捕获与丢包统计信息（tp_packets, tp_drops, tp_freeze_q_cnt 等）。

go
type PacketMMAPStats struct {
    Received uint32
    Dropped  uint32
    Freeze   uint32
}

Close

go
func (m *PacketMMAP) Close() error

卸载已映射的内存物理页并彻底关闭套接字描述符。
